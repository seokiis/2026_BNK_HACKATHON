# SERVICE.md — 서비스 정의서

> 이 문서는 AI 여신심사 어시스턴트의 서비스 정의입니다.
> Claude Code와 Copilot은 기능·화면·agent를 구현할 때 이 문서를 먼저 참조합니다.

---

## 1. 한 줄 정의

대출 상담 시 고객이 제출한 서류를 AI가 분석하여 여신 심사의견서 초안을
생성하는 서비스. 규정과 과거 심사 사례를 참고한 7명의 AI agent가 토론하여
9개 심사 항목별 의견을 자동 작성하고, 항목마다 상·중·하 진단과
데이터 부족·누락 여부를 제시한다.

## 2. 해결하려는 문제

영업점 직원이 작성한 심사요청서가 심사역에게 반려(빠꾸)되는 일이 잦다.
근거 부족, 항목 누락, 기준 미달을 사전에 걸러내지 못하기 때문이다.
이 서비스는 AI 토론으로 심사역 관점의 검증을 미리 수행해, 영업점 직원이
더 빠르고 일관성 있게 심사요청서를 작성하도록 지원하고 반려 횟수를 줄인다.

## 3. 사용자 (사람)

| 사용자 | 역할 |
|---|---|
| 영업점 대출 직원 (RM) | 고객 서류 등록, AI 초안 검토·편집·보강, 심사요청서 작성 |
| 심사역 | 영업점이 제출한 심사요청서 최종 심사 (이 서비스의 출력을 받는 쪽) |

> 주의: 위 2명은 "사람 사용자"다. 아래 7명의 AI agent와 혼동하지 말 것.
> 7명 agent는 사람을 돕는 내부 AI 페르소나다 (REVIEW_AGENTS.md 참조).

## 4. 전체 흐름

```
1. 영업점 직원이 고객번호 + 서류를 웹에 등록
2. 담당 심사역 자동 배정
3. AI 문서 분석(OCR)으로 기업·재무·계약 정보 구조화
4. 배정된 심사역의 과거 심사 사례 + 규정을 RAG로 검색
5. 7명 AI agent가 토론 (최대 3회 루프, 합의 충분 시 조기 종료)
6. 9개 항목별 심사의견 초안 생성 — 각 항목 상/중/하 + 데이터 부족·누락 라벨
7. 영업점 직원이 검토·편집, 필요 시 위험 보강·데이터 보강 버튼으로 외부자료 보강
8. 종합 의견 자동 생성
9. 심사요청서 양식으로 출력 → 여신시스템 DB 이관 → 심사역 제출
```

## 5. 핵심 특징

- 7명 AI agent의 토론 합의로 9개 항목을 동시 도출한다 (REVIEW_AGENTS.md).
- 상/중/하 진단의 중·하 항목은 마우스 오버 시 "위험 상태 보강" 버튼으로 바뀌어,
  외부 데이터 추적관이 외부자료를 검색해 보강 워딩을 생성한다.
- 데이터 부족·누락 항목은 마우스 오버 시 "데이터 보강" 버튼으로 바뀌어,
  외부자료를 검색해 초안을 보강한다.
- 생성된 의견에는 근거 문서와 인용 위치를 함께 표시한다.

## 6. 기술 방향 (요약 — 상세는 CLAUDE.md 6장)

- LLM 두뇌: Microsoft Foundry 배포 GPT 계열
- 지식·검색: Azure AI Search (규정·과거 심사사례·외부자료 RAG)
- 토론 오케스트레이션: 백엔드(FastAPI) 코드로 직접 제어 (프레임워크 미사용)
- 배포: Azure Container Apps

## 7. 금융 안전 원칙

- AI가 생성한 모든 의견은 "초안"이다. 최종 심사 결정은 심사역(사람)이 한다.
- 검색 근거에 없는 수치(금리·재무수치)를 agent가 지어내지 않는다.
- 모든 의견에 근거 문서를 표시한다.
- "AI가 판단한다" 표현 금지 → "AI가 지원한다".
