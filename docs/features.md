# FEATURES.md — 기능명세서

> 기능 구현 시 이 문서를 참조한다. 각 기능에 담당 개발 agent를 매핑했다.
> 9개 심사 항목은 REVIEW_ITEMS.md, 7명 토론 agent는 REVIEW_AGENTS.md 참조.

---

## 기능 목록

### 1. 기업 서류 업로드
- 입력: 사업자등록증, 재무제표, 기업소개서, 매출자료, CRETOP, 인포케어 등
- 형식: PDF, 이미지, Excel
- 추가: 고객번호 입력 → 담당 심사역 자동 배정
- 담당: frontend(업로드 UI) + backend(파일 수신·저장)

### 2. AI 문서 분석 (OCR)
- 업로드 문서를 분석하여 텍스트·핵심 정보 추출
- 출력: 기업정보·재무정보·계약정보 구조화
- 담당: ai-developer (OCR·추출), backend(저장)

### 3. AI 심사 (RAG)
- 규정 + 배정 심사역의 과거 심사 사례 기반 RAG 심사
- 규정 기반으로 정답 목데이터 생성·저장:
  소득금액증명원, 부가세 과세표준 증명원, 재무제표, 매출자료, 경매가액
  + 대출 실행된 사례의 심사의견
- 담당: ai-developer (RAG 인덱싱·검색)

### 4. 심사의견 자동 생성
- 추출 정보 기반 9개 항목 의견 초안 자동 생성
- 배정 심사역 타입에 맞춘 의견 생성
- 구현: 7명 토론 오케스트레이션 (REVIEW_AGENTS.md)
- 담당: ai-developer (토론 로직·LLM), backend(API·스트리밍)

### 5. AI 진단 (상·중·하)
- 각 항목을 규정·심사 기준으로 평가 (REVIEW_ITEMS.md 등급 기준)
- 중·하 항목: 마우스 오버 시 "위험 상태 보강" 버튼 → 외부 데이터 추적관이 외부자료로 보강
- 담당: ai-developer (등급·보강), frontend(호버 UI)

### 6. 데이터 상태 진단
- 작성 근거 부족·누락 탐지 (REVIEW_ITEMS.md 판정 기준)
- 부족/누락: 마우스 오버 시 "데이터 보강" 버튼 → 외부 데이터 추적관이 외부자료로 보강
- 담당: ai-developer (진단·보강), frontend(호버 UI)

### 7. 근거 데이터 표시
- 생성된 의견의 근거 문서·내용 함께 제공
- 출력: 문서 위치 + 인용 내용
- 담당: ai-developer (근거 추적), frontend(표시)

### 8. 심사의견 편집
- 사용자가 AI 작성 내용을 자유롭게 수정
- 수정에 따라 힌트 표시
  (예: "하반기 부채비율 30% 이상 예상" 작성 시 → "부채비율 30%는 무리일 것 같아요! 20%")
- 담당: frontend(편집 UI), ai-developer(힌트 생성)

### 9. (누락)
- 원본 명세서에 9번 없음. 의도된 결번.

### 10. 종합 의견 자동 생성
- 9개 항목을 종합하여 최종 심사의견 자동 작성
- 담당: ai-developer (종합 — 의장 agent), backend(API)

### 11. 심사요청서 출력
- 최종 심사의견을 심사요청서 양식으로 저장
- 9대 심사문장 + AI 등급(상/중/하) + 데이터 진단 결과를 하나의 패키지로 묶어
  여신시스템 DB에 최종 저장·이관
- 담당: backend(패키징·DB 이관), frontend(출력 UI)

### 12. 징구서류 안내
- 대출상품코드에 따른 서식 데이터 기반 징구서류 안내
- 담당: ai-developer(상품코드별 서식 검색), frontend(안내 UI)

### 13. 심사역 제출
- 심사역에게 전달
- 단, 상/중/하 및 데이터 부족·누락 라벨은 제외하고 전달
- 담당: backend(제출·라벨 제거)

## 화면 흐름 (디자이너·프론트 참조)

```
[서류 등록] → [AI 분석 진행] → [9개 항목 결과 + 등급/데이터 라벨]
   → [편집·보강] → [종합의견] → [심사요청서 출력] → [심사역 제출]
```
