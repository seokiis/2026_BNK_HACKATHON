## 0. 프로젝트 개요

BNK 해커톤(1박 2일)에서 개발하는 AI 기반 기업대출 여신심사 지원 시스템.
영업점 직원(RM)과 심사역이 기업대출 심사를 더 빠르고 정확하게 처리하도록 지원.
사업자 정보와 재무 데이터를 분석해 심사 초안을 자동 작성한다.

핵심 제약: Azure 사용 능력이 평가 기준. 메인 LLM은 Microsoft Foundry에 배포한
GPT 계열 모델. 외부 LLM(Claude API 등)을 런타임에 쓰지 않는다. 깃 레포 3종에서
배운 기술만 사용한다.

## 1. Think Before Coding

**Don't assume. Don't hide confusion. Surface tradeoffs.**

구현 전에:
- 가정을 명시적으로 말한다. 불확실하면 질문한다.
- 해석이 여러 개면 모두 제시한다. 조용히 하나 고르지 않는다.
- 더 단순한 방법이 있으면 말한다. 필요하면 반대한다.
- 불명확하면 멈춘다. 무엇이 헷갈리는지 짚고 질문한다.

## 2. Simplicity First

**문제를 푸는 최소 코드. 추측성 코드 금지.**

- 요청되지 않은 기능 추가 금지.
- 일회용 코드에 추상화 금지.
- 요청되지 않은 "유연성"·"설정 가능성" 금지.
- 불가능한 시나리오에 대한 에러 처리 금지.
- 200줄 짰는데 50줄이면 되면, 다시 쓴다.

스스로 물어라: "시니어 엔지니어가 이걸 과하게 복잡하다고 할까?" 그렇다면 단순화한다.

## 3. Surgical Changes

**필요한 것만 건드린다. 내가 만든 잔여물만 치운다.**

기존 코드 수정 시:
- 인접 코드·주석·포맷을 "개선"하지 않는다.
- 고장 안 난 걸 리팩터링하지 않는다.
- 내 방식과 달라도 기존 스타일을 따른다.
- 관련 없는 죽은 코드를 발견하면 언급만 한다. 삭제하지 않는다.

내 변경이 고아를 만들면:
- 내 변경으로 안 쓰이게 된 import·변수·함수만 제거한다.
- 요청 없이는 기존 죽은 코드를 제거하지 않는다.

기준: 변경된 모든 줄이 사용자 요청으로 직접 추적되어야 한다.

## 4. Goal-Driven Execution

**성공 기준을 정의한다. 검증될 때까지 반복한다.**

작업을 검증 가능한 목표로 변환:
- "검증 추가" → "잘못된 입력 테스트를 먼저 쓰고, 통과시킨다"
- "버그 수정" → "버그를 재현하는 테스트를 쓰고, 통과시킨다"
- "X 리팩터링" → "전후로 테스트가 통과하는지 확인한다"

다단계 작업은 간단한 계획을 먼저 말한다:
```
1. [단계] → 검증: [확인]
2. [단계] → 검증: [확인]
3. [단계] → 검증: [확인]
```

강한 성공 기준은 독립적 반복을 가능하게 한다. 약한 기준("작동하게 해줘")은 계속 확인이 필요하다.

> 이 지침이 작동하는 신호: diff에 불필요한 변경이 줄고, 과복잡으로 인한 재작성이 줄고,
> 실수 후가 아니라 구현 전에 명확화 질문이 나온다.
>
> **Tradeoff:** 이 지침은 속도보다 신중함에 치우친다. 사소한 작업은 판단에 맡긴다.

| Agent 파일 | 역할 | 호출 트리거 |
|---|---|---|
| `pm-architect.md` | 기획, 요구사항, 여신 프로세스 | "기획", "요구사항", "PRD", "화면 목록", "유저스토리" |
| `ui-designer.md` | 심사 UI/UX 설계 | "디자인", "화면 설계", "컴포넌트", "레이아웃" |
| `credit-expert.md` | 여신 실무 검증, 심사 로직 | "심사평 검토", "재무 분석 기준", "여신 로직", "컴플라이언스" |
| `ai-developer.md` | Azure AI Search + Foundry RAG | "검색", "인덱싱", "RAG", "임베딩", "에이전트 호출" |

## 7. 금융 도메인 안전 규칙 (전 agent 공통)

- AI 생성 심사평은 항상 "초안"이다. 최종 결정은 반드시 사람(심사역)이 한다.
- "AI가 판단한다"는 표현 금지 → "AI가 지원한다".
- 확정 수익·승인 보장 표현 금지.
- 투자·고위험 상품 언급 시 리스크 안내를 포함한다.
- 개인정보 처리 시 동의 절차를 전제한다.
- 검색된 문서에 없는 수치(금리·재무수치)는 임의 생성하지 않는다. 모르면 모른다고 한다.
