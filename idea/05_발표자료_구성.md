# 발표자료 구성안 — AI 여신심사 어시스턴트

> 클로드 디자인이 이 파일을 보고 슬라이드를 제작합니다.
> 각 섹션 = 슬라이드 1~2장 분량.

---

## 1. 오프닝 — 한 줄 소개

**"영업점 직원의 심사요청서 반려를 0에 가깝게 줄이는 AI 어시스턴트"**

- 서비스명: AI 여신심사 어시스턴트
- 핵심 가치: 7명 AI 전문가 토론 → 9개 심사항목 초안 자동 생성
- 대상: BNK 영업점 대출 직원(RM)

---

## 2. Pain Point — 왜 만들었는가

### 현재 문제
| 문제 | 구체적 상황 |
|---|---|
| 반려(빠꾸) 과다 | 영업점 직원이 심사역에게 제출 → 근거 부족·항목 누락·기준 미달로 반려 |
| 시간 낭비 | 심사요청서 1건 작성에 2~4시간, 반려 시 처음부터 다시 |
| 품질 편차 | 직원 경력·역량에 따라 심사요청서 품질 차이 극심 |
| 정보 접근 어려움 | 규정·과거 사례·외부 산업 데이터를 직원이 일일이 찾아야 함 |
| 심사역 과부하 | 품질 낮은 요청서를 반복 검토하느라 심사역 업무 과중 |

### 핵심 Pain Point (수치)
- 평균 반려율: 약 30~40% (영업점별 편차 큼)
- 반려 후 재작성 소요: 추가 1~2일
- 심사역 1인 처리 건수: 일 5~8건 → 반려 포함 시 실질 3~5건

---

## 3. 솔루션 — 어떻게 해결하는가

### Before vs After

| | Before (현재) | After (AI 어시스턴트) |
|---|---|---|
| 심사요청서 작성 | 직원이 수동으로 9개 항목 작성 (2~4시간) | AI가 초안 자동 생성 (5분) → 직원이 검토·편집 |
| 근거 확보 | 규정집·과거사례 수동 검색 | RAG가 자동으로 규정·사례 검색 + 근거 첨부 |
| 품질 검증 | 제출 후 심사역이 지적 (반려) | 제출 전 검증관(AI)이 미리 반박·보강 |
| 부족 데이터 | 직원이 눈치채지 못하면 반려 | AI가 "데이터 부족/누락" 자동 진단 + 보강 버튼 |
| 산업·외부 분석 | 직원이 뉴스·보고서 수동 조사 | 외부 데이터 추적관이 웹 검색·DART 자동 수집 |

### 기대 효과
- **반려율 30~40% → 5% 이하** (사전 검증으로 대부분 해소)
- **작성 시간 2~4시간 → 30분 이내** (AI 초안 + 편집)
- **품질 균일화** — 신입도 10년차 수준의 심사요청서
- **심사역 효율** — 고품질 요청서만 수신, 실질 처리량 40%↑

## 4. 동작 방식 — 서비스 플로우

### Step 1: 서류 등록
```
사업자번호 입력 → 기업 조회 + 담당 심사역 자동 배정
서류 업로드 (PDF/이미지) → OCR 자동 분석 → 데이터 구조화
여신 요청 정보 입력 (금액, 기간, 담보, 용도)
```

### Step 2: AI 7명 토론
```
┌─────────────────────────────────────────────┐
│  의장 — 개회, 서류·RAG 컨텍스트 배포          │
│  ↓                                           │
│  독립 분석 (3명 동시)                          │
│    · 수석 CPA → 재무 분석                     │
│    · 산업 애널리스트 → 업종·기술 분석           │
│    · 평판 리스크 탐지관 → 비재무 리스크         │
│  ↓                                           │
│  교차 평가 (2명)                              │
│    · 본부 수석 심사역 → 규정 적합성            │
│    · 기업영업 RM → 현장 관점                   │
│  ↓                                           │
│  검증관 반박 — 근거 부족·논리 허점 지적         │
│  ↓                                           │
│  재조사 (필요 시 루프, 최대 3회)               │
│  ↓                                           │
│  의장 합성 → 9개 항목 초안 생성               │
└─────────────────────────────────────────────┘
```
- 토론 과정을 실시간 스트리밍으로 직원에게 보여줌 (채팅 UI)
- 각 발언자 역할·아바타·발언 내용 표시

### Step 3: 결과 확인 & 편집
```
9개 항목 각각:
  [심사의견 텍스트] + [설득력 상/중/하] + [징구서류 충족/부족]
  + [근거 문서 칩] + [편집 가능]

위험 항목(중/하) → ⚡ 설득력 보강 버튼 → AI가 외부 데이터로 보강
서류 부족 항목 → 📥 서류 보강 버튼 → AI가 징구서류 안내
```

### Step 4: 제출
```
심사요청서 양식 출력 → 심사역에게 제출
(상/중/하 등급은 직원용 — 심사역에게는 전달하지 않음)
```

## 5. 핵심 기술 — Azure 기반 아키텍처

### 기술 스택 (전부 Azure)
| 계층 | 기술 | 역할 |
|---|---|---|
| LLM 두뇌 | Azure AI Foundry (GPT-4.1) | 7명 에이전트 페르소나별 추론 |
| 지식 검색 | Azure AI Search (하이브리드+시맨틱) | 규정·과거사례·외부자료 RAG |
| 임베딩 | text-embedding-3-large | 문서 벡터화 |
| 문서 분석 | Azure AI Document Intelligence | OCR + 구조화 추출 |
| 웹 검색 | Azure OpenAI Web Search (Responses API) | 산업 뉴스·동향 실시간 수집 |
| 저장소 | Azure Blob Storage | 서류 원본 저장 |
| 백엔드 | FastAPI (Python) on Container Apps | API + 토론 오케스트레이션 |
| 프론트 | React + TypeScript | 심사 UI |
| 인증 | Managed Identity + RBAC | 보안 |
| 배포 | Azure Container Apps + ACR | 운영 환경 |

### 차별점: 프레임워크 없는 직접 오케스트레이션
- LangChain, AutoGen 등 미사용
- 백엔드 코드로 7명 에이전트 토론 루프를 직접 제어
- 왜? → 금융 도메인 특수 로직(루프 조건, 검증관 반박 판정) 세밀 제어 필요

## 6. 데모 시나리오

### 시나리오: 한양정밀테크 (사업자번호 126-81-64926)
- 업종: 자동차 부품 제조 (금속 정밀가공)
- 내부신용등급: A
- 매출: 150억(2023) → 180억(2024) → 220억(2025) / CAGR 21.1%
- 총자산 180억, 부채 80억 → 부채비율 80% (양호)
- 쟁점: DSCR 1.18배 (내규 기준 1.25배 미달)
- AI 결론: **조건부 승인** — 수주확인서 반영 시 DSCR 1.31배로 기준 충족

### 데모 포인트
1. 사업자번호 입력 → 즉시 기업 정보 조회
2. 서류 업로드 → OCR 자동 분석
3. 7명 AI 토론 실시간 스트리밍 (채팅 형태)
4. 검증관이 DSCR 근거 미비 지적 → 수석 CPA 재조사 → 수주확인서 반영
5. 9개 항목 결과 — 상환재원 "하" 등급 표시
6. ⚡ 설득력 보강 클릭 → 외부 데이터로 DSCR 보강 워딩 자동 생성
7. 표 형태로 정리된 근거 데이터 확인
8. 심사요청서 출력 & 제출

### 기술적 근거
- Azure AI Foundry: 기업용 LLM — 보안·규정 준수·SLA 보장
- 직접 오케스트레이션: 금융 규정 로직을 코드로 정밀 제어 → 프레임워크 종속 없음
- RAG: 규정 업데이트 시 인덱스만 갱신 → 모델 재학습 불필요
- 근거 추적: 모든 의견에 출처 첨부 → 감사 추적(Audit Trail) 가능

## 8. 금융 안전 & 컴플라이언스

- **AI는 "초안 지원"만 한다** — 최종 결정은 반드시 심사역(사람)
- 모든 의견에 근거 문서 첨부 — 검증 가능
- 수치를 임의 생성하지 않음 — 데이터 없으면 "누락" 표시
- "AI가 판단" 표현 금지 → "AI가 지원"
- 개인정보 동의 절차 전제
- Azure 기업용 보안 (Managed Identity, RBAC, 데이터 국지화)

## 10. 클로징

**"반려 없는 심사요청서, AI가 만들어 드립니다"**

- 7명 AI 전문가가 심사역처럼 미리 검증

- 영업점 직원은 검토·편집만 → 심사역은 고품질 요청서만 수신
- **Azure 올인원** — Foundry + AI Search + Container Apps
- **금융 안전** — AI는 초안, 사람이 최종 결정
