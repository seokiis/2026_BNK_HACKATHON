> 대화 일시: 2026-06-20 01:17  
> 참여: 사용자 + AI

---

## 1. 배경 / 문제

- 현재 웹(bnk-ui.html)에는 **창원정밀** 예시만 들어있음
- 학습에 사용하지 않은 **새로운 기업 데이터**를 넣어서 실제 동작을 보여줘야 함
- 재무적으로 대출 반려 판정이 나더라도, **산업현황·뉴스·시장 트렌드**를 조사해
  "그래도 대출이 가능한 근거"를 보완할 수 있어야 함

### 핵심 시나리오 예시

> 재무: 부채비율 220% (내규 200% 초과 → 반려)  
> 그러나: 해당 산업이 호황기, 최근 3년 매출액 지속 확대, 현금흐름 양호  
> → AI가 산업현황 근거를 자동 조사해서 "조건부 승인" 근거로 제시

## 2. 필요한 것들

### A. 데모용 입력 데이터 (Input)

현재 `data/sample_company_data.json` 구조 그대로, 새 기업을 추가:

| 시나리오 | 기업명(가칭) | 특징 |
|---------|-------------|------|
| 재무 양호 → 승인 | 한양테크 | 부채비율 80%, ICR 3.5x, 매출 성장 |
| 재무 불량 + 산업 호황 → 조건부 승인 | 동진소재 | 부채비율 250%, 그러나 반도체 소재 산업 호황 |
| 재무 불량 + 산업 침체 → 반려 | 삼원건설 | 부채비율 300%, 건설업 침체기 |
| 재무 경계선 + 뉴스 긍정 | 남해바이오 | 부채비율 195%, 바이오 정부 지원 확대 뉴스 |

### B. 산업현황 / 뉴스 데이터

AI가 "산업이 호황이다"라고 판단하려면 근거 데이터가 필요:

1. **산업 동향 JSON** — 업종코드별 호황/보통/침체 + 근거 (BSI지수, 수출통계 등)
2. **뉴스 스니펫** — 업종 관련 최근 뉴스 제목+요약 (모의 데이터 또는 실제 크롤링)
3. **시장 전망 리포트** — 증권사/연구원 전망 요약

```json
// 예시: industry_trends.json

  "C26": {
```

## 3. AI 보완 로직 흐름

```
[입력 데이터] → [내규 규칙 적용] → 반려 판정?
                                       │
                                       ▼ YES
                            [산업현황 조회 (Azure AI Search)]
                                       │
                            [관련 뉴스 검색 (Azure AI Search)]
                                       │
                                       ▼
                            [보완 근거 생성 (Foundry GPT)]
                                       │
                            "부채비율 초과이나, 반도체 산업 호황으로
                             매출 확대 추세. 현금흐름 양호하여
                             조건부 승인 가능성 검토 필요"
                                       │
                                       ▼
                            [최종 심사의견서에 반영]
```

### 7명 토론에서의 역할 분담

- **산업분석가** agent: 산업현황 데이터 기반 의견 제시
- **리스크관리자** agent: 재무 위험 강조
- **토론 결과**: 양쪽 근거를 모두 담아 심사역에게 판단 위임

## 4. 구현 방식: React vs 순수 HTML?

### 결론: **기존 React 프론트 사용 권장** (HTML은 데모 빠른 확인용만)

| 비교 | React (기존 프론트) | 순수 HTML |
|------|-------------------|-----------|
| 해커톤 평가 | ✅ 완성도 높음 | ❌ 프로토타입 느낌 |
| API 연동 | ✅ 이미 구조 있음 | ⚠️ fetch로 가능하나 복잡 |
| 상태 관리 | ✅ 여러 기업 전환 용이 | ❌ 수동 DOM 조작 |
| 토론 과정 표시 | ✅ 실시간 스트리밍 UI | ⚠️ 가능하나 코드 지저분 |
| 개발 속도 | ⚠️ 컴포넌트 추가 필요 | ✅ 빠르게 한 장 가능 |

### 추천 전략

1. **빠른 검증** → `idea/demo.html`로 API 호출 테스트 (10분)
2. **실제 데모** → React 프론트에 "기업 선택" 드롭다운 + 산업분석 탭 추가
3. **발표용** → React로 통합된 화면 (심사의견서 + 산업근거 패널)
